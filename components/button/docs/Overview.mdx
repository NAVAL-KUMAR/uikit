# Variants
## Primary Button

To call attention to an action on a form, or highlight the strongest call to action on a page. Primary buttons should only appear once per screen.

**Example : Save, Confirm, Done**

**Sequence: Default > Hover > clicked > Disabled**

<Documenter component={props.components}>

```preview
return (
  <Button type="primary">Button Label</Button>
)

```

</Documenter>

## Secondary Button

For Secondary actions on the page. Use against shaded or colorful backgrounds. An Secondary button will maintain the appropriate visual weight and wonâ€™t clash with the background color and maintain the visual hierarchy. 

**Example:**

**Sequence: Default > Hover > clicked > Disabled**

<Documenter component={props.components}>

```preview

  <Button type="secondary">Button Label</Button>
)

```

</Documenter>

## Tertiary Button

For least insignificant actions on the page. They can be paired with primary and secondary button when there are multiple calls to action. 

**Example:**

**Sequence: Default > Hover > clicked > Disabled**

<Documenter component={props.components}>

```preview

  <Button type="tertiary">Button Label</Button>
)

```

</Documenter>

## Warning Button

Warning button is used when user is asked to take an action, usually during the final confirmation for a destructive or negative action. They are generally used mostly in confirmation modals. 

**Example :  Delete**

**Sequence: Default > Hover > clicked**

<Documenter component={props.components}>

```preview

  <Button type="warning">Button Label</Button>
)

```

</Documenter>
